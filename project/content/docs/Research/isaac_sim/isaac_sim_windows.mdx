## Isaac sim與Isaac Lab的比較

兩者皆是Nividia Omniverse旗下的產品，但Isaac Lab是基於Isaac Sim之上一個更輕量、高階的套件。以下是兩者比較表：

| **項目** | **Isaac Sim** | **Isaac Lab** |
| --- | --- | --- |
| **定位** | 通用機器人模擬器平台 | 建立於 Isaac Sim 之上的 訓練框架 |
| **主要用途** | 物理模擬、場景建置、感測器模擬 | 強化學習、訓練、環境管理 |
| 下載難易度 | 較簡單 | 較複雜 |
| 開發彈性 | 較高 | 較低 |
| 上手難易度 | 因為Isaac Sim給予開發者更高的自由度，所以更加新手不友善 | 相對Isaac Sim簡單 |
| 顯示軟體 | Isaac Sim | Isaac Sim |

## 連線方式

### 1. 登入伺服器

因為**Isaac Sim**需要很高的算力，所以建議在GPU上操作

打開電腦的終端機，並輸入：

```bash
ssh username@hostname -p port_number
```

接著會再要求輸入密碼，因為加密的關係，所以輸入密碼時不會顯示任何輸入資料，打完之後直接按下Enter即可，登入後開頭會出現$的字樣及算成功。登入後可以輸入`nvidia-smi` 確認顯卡狀態：

![isaac sim.png](/isaac sim.png)

### 2. 安裝 Docker 與 NVIDIA 環境

接著為了不破壞環境，因此建議可以安裝Docker。Docker 是一種容器化技術，透過隔離行程與檔案系統來建立獨立的執行環境，因為這次機器狗的作業系統ROS2 版本間與Ubuntu版本有極度依賴性，例如ROS2 Jazzy 綁定Ubuntu  24.04，ROS2 Humble綁定Ubuntu  22.04，如果不使用Docker的話，則電腦的Ubuntu也會連帶不可以更新，否則整個專案都會遭受影響。

先安裝Docker：

```bash
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh
```

為了讓Docker可以使用GPU，所以也需要下載NVIDIA Container Toolkit：

```bash
curl -fsSL [https://nvidia.github.io/libnvidia-container/gpgkey](https://nvidia.github.io/libnvidia-container/gpgkey) | sudo gpg --dearmor -o /usr/share/keyrings/nvidia-container-toolkit-keyring.gpg
curl -s -L [https://nvidia.github.io/libnvidia-container/stable/deb/nvidia-container-toolkit.list](https://nvidia.github.io/libnvidia-container/stable/deb/nvidia-container-toolkit.list) | \
sed 's#deb https://#deb [signed-by=/usr/share/keyrings/nvidia-container-toolkit-keyring.gpg] https://#g' | \
sudo tee /etc/apt/sources.list.d/nvidia-container-toolkit.list
sudo apt-get update && sudo apt-get install -y nvidia-container-toolkit
sudo nvidia-ctk runtime configure --runtime=docker
sudo systemctl restart docker
```

### 3. 取得並設定 NVIDIA NGC 金鑰

再來還需要取得 NVIDIA NGC 專屬金鑰 (API Key)才可以在Docker上拉取NVIDIA的Image
1.先前往NVIDIA NGC 官方網站：[ngc.nvidia.com](https://ngc.nvidia.com/)註冊開發者帳號
2.點擊右上角的使用者頭像，選擇 **Setup
3.**點擊 **Generate API Key
4.取得 API Key 後，回到終端機，使用 Docker 登入 NVIDIA NGC**

```bash
docker login nvcr.io
```

**Username:** 輸入 `$oauthtoken` 
**Password:** 貼上你剛剛在 NGC 網站上生成的 API Key

### 4. 下載與啟動 Isaac Sim 環境

接著把把 Isaac Sim 的 Image（Pull）下來(至於後面的版本要先確認)

```bash
docker pull nvcr.io/nvidia/isaac-sim:4.5.0-ros2-humble-1.0
```

啟動環境：

```bash
mkdir -p ~/isaac-sim/cache/kit \
~/isaac-sim/cache/ov \
~/isaac-sim/cache/pip \
~/isaac-sim/cache/glcache \
~/isaac-sim/cache/computecache \
~/isaac-sim/logs \
~/isaac-sim/data
```

```bash
docker run --name isaac-sim --entrypoint bash -it --gpus all --dns 8.8.8.8 \
-e "ACCEPT_EULA=Y" -e "PRIVACY_CONSENT=Y" \
-p 8211:8211 \
-p 8899:8899 \
-p 48010:48010 \
-v ~/isaac-sim/cache/kit:/isaac-sim/kit/cache:rw \
-v ~/isaac-sim/cache/ov:/root/.cache/ov:rw \
-v ~/isaac-sim/cache/pip:/root/.cache/pip:rw \
-v ~/isaac-sim/cache/glcache:/root/.cache/nvidia/GLCache:rw \
-v ~/isaac-sim/cache/computecache:/root/.nv/ComputeCache:rw \
-v ~/isaac-sim/logs:/root/.nvidia-omniverse/logs:rw \
-v ~/isaac-sim/data:/root/isaac-sim/data:rw \
nvcr.io/nvidia/isaac-sim:4.5.0-ros2-humble-1.0
```

### 5. 畫面傳輸與檢視

接著大致後端流程就完成了！要來做畫面傳輸的工作，讓前端Windows可以看到畫面，其實也可以採無頭模式開發(沒有畫面)，但因為當初我們認為有畫面比較可以實際也比較不抽象。有兩種畫面呈現方式：

1. 直接用瀏覽器開（新版支援）：`http://GPU的IP:8211/streaming/client/`
2. 下載 **NVIDIA Omniverse Streaming Client** 桌面程式連線 → 我們選用這種
(Omniverse Streaming Client 目前已停止更新，若無法下載建議改用瀏覽器方式連線。)

下載完應用程式後，先回到終端機docker畫面(提示符為 `root@...`)，執行：

```bash
# 1. 載入 ROS 2 環境 (因為是存檔版，所以檔案一定在)
source /opt/ros/humble/setup.bash

# 2. 設定通訊協定 (這行一定要打，環境變數重開會重置)
export RMW_IMPLEMENTATION=rmw_cyclonedds_cpp

# 3. 啟動直播
./runheadless.webrtc.sh
```

再打開應用程式，輸入Ubuntu帳號的IP位址，即可看到畫面
